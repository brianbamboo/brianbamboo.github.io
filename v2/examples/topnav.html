So this style of top navigation is cool. Basically the way it works is by classing and declassing the topnav bar as responsive whenever the icon is clicked. When the topnav bar is classed as responsive, the positioning of the topnav element is set to "relative" and each of the .a children in the topnav are set to float:none, display:block, and text-align:left, except for the icon, which is set as position:absolute in the top right corner.

One problem with this style of top navigation, though, is that the topnav bar doesn't automatically re-class to topnav when the screen size becomes wider. This is a minor detail, but it bothers me.
