An analysis of public Wi-Fi in NYC¶

Today, I'll be taking a look at an NYC Wi-Fi hotspots dataset obtained from Kaggle here: https://www.kaggle.com/new-york-city/nyc-public-wifi. This dataset, uploaded by the City of New York, lists every public Wi-Fi hotspot in New York City.

I'm a bit curious to how the city determines what's considered a "public" hotspot, so I'll make a mental note to do some research into that later. First, let's load the data and take a look at the variables included in the dataset, along with what they mean. The dataset also comes with a data dictionary that should also help with this.

I'll also note that I do most of my work in R, so this is my first time trying to type a quick analysis in Python. It'll be some good practice getting used to some of the different features in Python and seeing how they are different than or similar to their R counterparts.

I will say that at a first go, visualization in Python is a little bit more difficult / unintuitive than data visualization in R. ggplot2 has really developed a strong, intuitive grammar for building graphics that Python doesn't seem to be able to compete with. The boroughs, ranked from greatest to least in terms of number of Wi-Fi hotspots, are: Manhattan, Brooklyn, Queens, the Bronx, and Staten Island. Manhattan has 1,204 Wi-Fi hotspots, Brooklyn has 595 Wi-Fi hotspots, Queens has 415 Wi-Fi hotspots, the Bronx has 257 Wi-Fi hotspots, and Staten Island has 95.

Seeing this, a few questions that come to mind are:

  • How does the number of hotspots relate to borough population (residential)?
  • How does the number of hotspots relate to wealth of borough residents?
  • How does the number of hotspots relate to borough labor force?

I do have Zip Code data in this dataset, so I'll place these questions at the bottom of the notebook for now to return to when I finish up with visualizations. Ideally, I'll be able to join some demographic data and see if any of them are correlated with the number of Wi-Fi hotspots. We'll probably have to do it at the Zip Code level, though, because if we do it at the borough level I'll have a sample size of only 5 boroughs which isn't too promising.

We also see that most 1983 Wi-Fi hotspots are free, 581 are limited free, and 2 hotspots in this dataset are partner site hotspots. The multiple linear regression with population, median, average, and per capita income as predictors is able to explain about 50% of the variation in the response, which is pretty promising. For a zip code, it turns out that per capita income is a pretty damn strong predictor of how many Wi-Fi hotspots are in an area. This makes sense - I feel like per capita income is often the best measure of the overall wealth in an area. Every 10,000 increase in per capita income in a zip code is associated with approximately 5 Wi-Fi hotspots.

In this regression, the coefficients for median and average household income aren't significantly different from zero statistically.

The coefficient for population is still significantly different from zero, with a 10,000 increase in population approximately associated with about 2 additional Wi-Fi hotspots.
